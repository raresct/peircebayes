Command line options¶

usage: peircebayes [-h] [-n N] [-b BURN] [-l LAG] [-s SEED] [-t] [-d] [-c]
                   [-m {joint,lda_ll}] [-a {ungibbs,amcmc}]
                   in_file
Positional arguments:
in_file input pb file
Options:
-n=100 number of samples
-b=0, --burn=0 number of burn-in iterations
-l=1, --lag=1 keep samples every l iterations (l=1 means no lag)
-s=1234, --seed=1234
 seed for numpy rng
-t=False, --track=False
 include metric information (see -m)
-d=False, --debug=False
 include some debug information
-c=False, --config=False
 load arguments from json as in_file
-m=joint, --metric=joint
 

choose metric to track

Possible choices: joint, lda_ll

-a=ungibbs, --algo=ungibbs
 

choose sampling algorithm. ‘ungibbs’ is uncollapsed gibbs sampling on x and theta ‘amcmc’ is ancestral mcmc, i.e. sample x and theta is E[Dir(alpha+x)]

Possible choices: ungibbs, amcmc
